# Карта title / description / H1
## Проект: Крымская Кровельная

## 1. Правила meta-данных
- у каждой индексируемой страницы свой уникальный `title`, `description` и `H1`;
- `H1` один на страницу;
- `title` должен закрывать основной интент без переспама;
- `description` — усиливать CTR, а не повторять `title`;
- коммерческие страницы по возможности содержат ориентир цены или ключевое обещание;
- бренд добавляется там, где это усиливает доверие, но не обязан повторяться на всех URL одинаково.

## 3. Дополнительные правила

### 3.1. Для разделов
Разделы не должны быть пустыми хабами. В них обязательно:
- краткое объяснение направления;
- карточки дочерних услуг;
- блок преимуществ;
- FAQ;
- CTA.

### 3.2. Для кейсов
В `title` кейсов желательно сочетать:
- тип работ;
- тип объекта или проблему;
- географию, если она усиливает доверие.

### 3.3. Для статей
В информационных статьях не использовать чисто рекламные titles. Они должны:
- обещать ответ на вопрос;
- быть понятными человеку;
- вести в связанную услугу через структуру страницы, а не через переспам в title.

### 3.4. Для geo-страниц
Geo-страницы не должны копировать главную и услуги с заменой только названия города.
