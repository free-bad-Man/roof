# Правила индексации
## Проект: Крымская Кровельная

## 1. Базовый принцип индексации
Индексируем только страницы, которые:
- закрывают реальный спрос;
- имеют самостоятельную ценность;
- содержат уникальный HTML-контент;
- не являются дублями других URL;
- не созданы ради расширения дерева без смысла.

Не индексируем всё, что:
- служебное;
- пустое;
- тестовое;
- дублирующее;
- параметрическое;
- тонкое по ценности.

---

## 2. Матрица индексации

| Тип страницы | Индексация | Условия |
|---|---|---|
| Главная | index | Всегда |
| Разделы услуг | index | Если наполнены и ведут в дочерние услуги |
| Страницы услуг | index | Если уникальны по интенту и содержанию |
| Кейсы | index | Если это полноценные разборы объектов, а не фотогалереи |
| Статьи | index | Если несут самостоятельную пользу и связаны с услугами |
| О компании | index | Всегда |
| Контакты | index | Всегда |
| Geo-страницы | index | Только при уникальном локальном контенте |
| Поиск по сайту | noindex / disallow | Всегда |
| Thank-you | noindex / disallow | Всегда |
| Тестовые страницы | noindex / disallow | Всегда |
| URL с параметрами | canonical на чистый URL, при необходимости noindex | По умолчанию не индексировать |
| Архивы тегов / рубрик, если появятся | noindex | Если не несут самостоятельной ценности |
| Дубли со слешем / без слеша | 301 | Всегда |
| Дубли http / https | 301 на https | Всегда |
| Дубли с www / без www | 301 на без www | Всегда |

---

## 3. Canonical

### 3.1. Общие правила
- у каждой индексируемой страницы должен быть self-canonical;
- canonical всегда указывает на итоговый чистый URL;
- canonical ставится только на 200 OK страницу;
- canonical не должен указывать на URL с параметрами;
- canonical не заменяет 301, если есть полноценный дубль.

### 3.2. Формат canonical
Единый вид:
- `https://domain.ru/krovelnye-raboty/remont-krovli/`

### 3.3. Когда нужен 301, а не только canonical
301 обязателен для:
- `http` → `https`
- `www` → без `www`
- без слеша → со слешем
- дублей регистра
- старых URL после изменения структуры

---

## 4. Robots

### 4.1. Базовый шаблон
```txt
User-agent: *
Disallow: /wp-admin/
Disallow: /search/
Disallow: /?s=
Disallow: /thank-you/
Disallow: /test/
Clean-param: utm_source&utm_medium&utm_campaign&utm_term&utm_content&ysclid&yclid
Sitemap: https://site.ru/sitemap.xml
```

### 4.2. Что закрываем
- поиск;
- служебные страницы;
- тестовые URL;
- thank-you страницы;
- технические директории CMS;
- параметрический мусор.

### 4.3. Что не закрываем
- главную;
- разделы услуг;
- страницы услуг;
- кейсы;
- статьи;
- контакты;
- о компании;
- geo-страницы, если они реально участвуют в индексации.

### 4.4. Важное правило
`robots.txt` не должен блокировать то, что нужно индексировать и что участвует в передаче сигналов качества.

---

## 5. Sitemap

### 5.1. В sitemap включаем только
- canonical URL;
- страницы с 200 OK;
- страницы без `noindex`;
- страницы, реально участвующие в индексации.

### 5.2. Логическая структура sitemap
- основная карта сайта;
- при росте проекта допустимо дробление на:
  - `sitemap-pages.xml`
  - `sitemap-services.xml`
  - `sitemap-cases.xml`
  - `sitemap-articles.xml`
  - `sitemap-geo.xml`

### 5.3. Что не включаем
- благодарности;
- тестовые страницы;
- search;
- дубли;
- параметрические URL;
- слабые geo-страницы, которые не прошли порог уникальности.

---

## 6. Правила для geo-страниц

### 6.1. Когда geo-страница допустима
Geo-страница публикуется только если есть минимум:
- уникальный локальный вступительный блок;
- уникальный FAQ по городу;
- отдельные кейсы по городу или близкой зоне;
- локальные фото, детали работы или особенности выезда;
- собственная перелинковка из контактов, главной и кейсов.

### 6.2. Когда geo-страница недопустима
Не публикуется, если:
- меняется только название города;
- нет кейсов и локального доказательства;
- страница повторяет главную или страницу услуги;
- нет коммерческой ценности.

### 6.3. Гео-правило проекта
На старте индексируем:
- `/simferopol/`

`/sevastopol/` и `/yalta/` допускаются позже, но не автоматически.

## 7. Микроразметка

### 7.1. На всём сайте
- `WebPage`
- `BreadcrumbList`

### 7.2. Для компании
- `Organization` или `LocalBusiness`
- `PostalAddress`
- `telephone`
- `email`

### 7.3. Для контактов
Обязательно:
- название компании
- адрес
- телефон
- email
- режим работы
- география обслуживания

### 7.4. Для FAQ
`FAQPage` ставится только там, где FAQ реально выведен в HTML и не является скрытым дублирующим блоком.

### 7.5. Для кейсов и статей
Допустимо использовать:
- `Article` для статей;
- `WebPage` для кейсов, если отдельная схема кейса не реализуется.

### 7.6. Для услуг
Если будут использоваться коммерческие элементы, разметка не должна вводить в заблуждение. На старте достаточно:
- `WebPage`
- `BreadcrumbList`
- `FAQPage`, если есть FAQ

---

## 8. H1, Title, Description
- один `H1` на страницу;
- уникальный `title`;
- уникальный `description`;
- без переспама одинаковыми словоформами;
- без автогенерации одного и того же шаблона на десятки URL.

---

## 9. Технические правила качества индексируемой страницы
Каждая индексируемая страница должна:
- отдавать `200 OK`;
- быть доступной без авторизации;
- иметь читаемый HTML-контент;
- не прятать основной текст в JS;
- иметь canonical;
- быть включённой в sitemap;
- иметь внутренние ссылки с других страниц;
- нормально читаться на мобильных устройствах.

---

## 10. Страницы и блоки, которые должны быть в HTML
Нельзя прятать от индексации или отдавать только через JS:
- основной заголовок;
- коммерческий оффер;
- описание услуги;
- цены «от»;
- FAQ;
- географию;
- контакты;
- кейсы и анонсы кейсов;
- основные CTA.

---

## 11. Мягкие noindex-правила на будущее
Если на сайте появятся новые сущности, они не индексируются автоматически.

Сначала проверяется:
- есть ли спрос;
- есть ли уникальный интент;
- есть ли уникальное содержание;
- не перекрывает ли страницу уже существующий URL.

Если ответа нет — страницу либо не создаём, либо оставляем вне индексации.

---

## 12. Контрольный чек-лист перед публикацией
1. URL чистый и финальный
2. Страница отдаёт 200
3. Есть self-canonical
4. Есть уникальные title / description / H1
5. Нет дубля существующей страницы
6. Есть внутренняя перелинковка
7. Контент в HTML
8. Страница есть в sitemap
9. Страница не закрыта robots или meta noindex
10. На странице есть реальная ценность, а не SEO-заглушка
